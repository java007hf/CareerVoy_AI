# CareerVoy AI - PRD（需求文档）

## 1. 产品概述

### 1.1 产品定位
CareerVoy AI 是一个面向失业人群与在职副业人群的「社区 + 搞钱」平台。

**平台形态**：首期为 Web 应用，通过浏览器访问，采用响应式设计适配桌面端和平板端。

核心目标是：
- 提供高实用价值的再就业/副业信息；
- 提供情绪支持与同伴连接；
- 提供可落地、可追踪的带货变现路径（首期聚焦淘宝客）。

### 1.2 愿景
让用户在职业不确定阶段，既能获得有用信息与情绪支撑，也能更快建立稳定收入来源。

---

## 2. 产品目标（Why）

### 2.1 业务目标
1. 构建失业/副业人群高活跃社区，形成高质量经验沉淀。
2. 搭建从内容种草到带货转化的闭环，提高用户变现效率。
3. 打造"信息 + 情绪 + 收入"三位一体的平台心智。

### 2.2 用户价值目标
1. 快速获取可执行信息（如面试技巧、失业金流程、避坑指南）。
2. 获得情绪共鸣与互助支持，缓解焦虑与孤独感。
3. 获取副业变现机会，看到真实收益案例与方法。

### 2.3 首期建议核心指标（待确认）
- 社区侧：
  - D7 留存率
  - 人均发帖/互动次数
  - 高质量内容占比（被收藏/被点赞/完读率）
- 变现侧：
  - 带货内容发布人数
  - 点击-下单转化率
  - 人均带货收入
- 平台侧：
  - DAU/MAU
  - 举报率与违规内容率（风控健康度）

---

## 3. 目标用户（Who）

### 3.1 核心用户群
1. **失业人群**：
   - 刚离职或长期待业；
   - 急需就业信息、政策流程、短期收入机会；
   - 需要情绪支持和同伴网络。

2. **在职副业人群（第二职业探索者）**：
   - 白天上班，晚间/周末尝试副业；
   - 关注低门槛、可复制、可持续的变现方式；
   - 愿意分享经验并寻找合作资源。

### 3.2 用户典型痛点
- 信息碎片化，真假难辨，不知从何入手；
- 副业教程"看起来都能做，实际难落地"；
- 缺少可信的同路人反馈与陪伴；
- 变现数据不透明，看不到真实案例与收益路径。

## 4. 产品范围（What）

首期包含三大模块：
1. 社区
2. 搞钱（首期淘宝客）
3. 个人信息中心

### 4.1 社区模块
社区承载四类核心内容：

1. **实用信息内容**
   - 面试技巧
   - 失业金领取流程
   - 职业避坑指南

2. **情绪共鸣内容**
   - 失业后的日常生活分享
   - 心路感悟
   - 解压方法与恢复建议

3. **盈利相关内容**
   - 带货选品推荐
   - 佣金分成比例说明
   - 销售战报（成绩复盘、案例拆解）

4. **同城互助内容**
   - 线下互助活动
   - 面基邀请
   - 即时求助信息

#### 4.1.1 社区基础功能（MVP）
- 帖子发布（支持图文，提供图片上传入口）
- 帖子浏览（推荐流/最新流/关注动态流）
- 帖子搜索（支持关键词搜索社区内容）
- 点赞、评论、收藏、转发（待确认是否首期全开）
- 评论内点赞与回复（支持对评论进行点赞和回复，形成二级互动）
- 关注用户
- 话题/分类标签
- 举报与内容审核基础机制

### 4.2 搞钱模块（首期：淘宝客）

#### 4.2.1 目标
帮助用户完成从选品到分销再到收益追踪的带货闭环。

#### 4.2.2 核心能力（MVP）
- 商品/选品信息展示（标题、佣金比例、预计收益等）
- 商品筛选与排序能力（按分类、佣金、销量等）
- 商品详情页（券后价、佣金详情、推广入口、商品图文介绍）
- 推广素材与文案参考（待确认）
- 带货链接生成与分享（待确认接入方式）
- 基础战报展示（点击、订单、预估佣金）
- 商品搜索（支持关键词搜索商品池）

#### 4.2.3 搞钱模块子功能
搞钱模块内部通过左侧边栏进行二级导航，包含以下功能：

- **选品广场**: 核心功能，用于浏览、筛选、发现所有可供推广的商品。这是用户"选货"的地方。
- **我的推广**: 用户的"推广资产"管理页。这里会收录所有用户已经生成了推广链接的商品，方便他们：
    1.  **快速查找**：快速找到自己主推的几个商品。
    2.  **复制链接/口令**：无需重新生成，方便再次分享。
    3.  **查看单品效果**：展示每个推广商品的引流（点击量）与转化（订单、佣金）数据，帮助用户判断哪个品更受欢迎。
- **收益看板**: 用户的"财务中心"，聚合展示所有推广活动产生的收入数据，包括：
    1.  **核心数据统计**：本月预估、上月结算、累计结算等。
    2.  **收益明细**：每一笔有效订单的佣金来源。
    3.  **提现记录与操作**：展示历史提现状态，并发起新的提现。
- **新手入门**: 这是一个静态的"帮助中心"或"教程区"，专门为新用户准备，内容可以包括：
    1.  **平台规则**：讲解平台的带货、结算、社区规范。
    2.  **操作指南**："第一步做什么、第二步做什么"的流程教学。
    3.  **常见问题（FAQ）**：例如"佣金怎么算？""提现多久到账？"等。
    4.  **优秀案例**：分享高收入用户的成功经验。

#### 4.2.4 后续扩展方向
- 从淘宝客扩展至更多品类与平台（如本地生活、知识付费、服务分销等）。

### 4.3 个人信息模块

#### 4.3.1 用户资产与行为管理
- 我的帖子
- 消息通知
- 我的关注
- 带货收入（预估/结算）

#### 4.3.2 账号基础能力（MVP）
- 登录（手机号/微信）
- 登出
- 用户个人设置（头像、昵称、地点、个人说明等）
- 内容管理（删除/编辑自己的帖子，编辑需在发布后限定时间内操作）
- 消息中心（互动消息、系统通知、带货通知分类展示）

### 4.4 全局功能

#### 4.4.1 导航结构
- Web 端顶部导航栏：品牌 Logo + 搜索入口 + 消息通知 + 用户头像
- Web 端左侧边栏一级导航：社区 / 搞钱 / 消息 / 我的
- 搞钱模块内部通过左侧边栏进行二级导航（选品广场 / 我的推广 / 收益看板 / 新手入门）
- 整体为 Web 端布局风格，参考 `参考设计/` 目录下的设计风格

#### 4.4.2 用户主页
- **我的主页**：展示个人信息、我的帖子、我的关注/粉丝数
- **他人主页**：查看他人帖子、关注入口

#### 4.4.3 空状态设计
- 各列表页（帖子流、商品列表、消息等）无内容时展示引导信息，帮助新用户理解功能并促发首次操作

## 5. 用户关键场景（When/Where）

1. 用户刚失业，急需了解失业金领取流程与再就业策略。
2. 用户希望下班后做副业，寻找"可马上执行"的带货方法。
3. 用户发布自己的带货战报，吸引关注并建立个人影响力。
4. 用户希望在同城找到可以互相支持/合作的人。
5. 用户每天查看自己的消息与带货收入变化。
6. 用户搜索某个面试技巧或某类商品，快速找到所需内容。
7. 用户进入他人主页，浏览其历史帖子并决定是否关注。

## 7. 核心流程（How）

### 7.1 新用户首日路径（建议）
1. 注册登录
2. 选择关注方向（找工作 / 副业搞钱 / 情绪互助）
3. 浏览社区精选内容
4. 查看"搞钱"模块选品
5. 发布首条动态或收藏首个带货案例

### 7.2 带货变现路径（淘宝客）
1. 用户进入"搞钱"频道
2. 选择商品并查看佣金比例、案例战报
3. 生成推广链接并分发
4. 在"个人中心"查看点击/订单/预估收入

### 7.3 社交互动路径
1. 用户浏览帖子，对评论点赞或回复
2. 进入他人主页，查看历史帖子
3. 关注用户后，在"关注动态流"中持续看到其新内容

## 8. 风险与治理（首期必须考虑）

1. **内容真实性风险**：
   - 虚假战报、夸大收益、误导性教程。
2. **合规风险**：
   - 招聘/金融/医疗等敏感信息违规发布。
3. **社区氛围风险**：
   - 人身攻击、焦虑贩卖、诈骗导流。
4. **交易与分佣争议风险**：
   - 收益口径不一致引发纠纷。

### 8.1 基础治理策略（建议）
- 举报 + 机器审核 + 人工复审；
- 敏感词与高风险外链拦截；
- 收益数据增加"预估/结算"状态说明；
- 建立社区公约与违规处罚梯度。

## 13. 淘宝客接入与分成可行方案（首期）

### 13.1 业务目标
跑通"选品 -> 推广 -> 订单回流 -> 收益展示 -> 结算"的闭环。

### 13.2 用户带货流程
1. 用户在"搞钱"模块浏览商品池（券后价、佣金比例、预估收益）。
2. 用户选择商品并生成专属推广链接/口令。
3. 用户在外部渠道分发（社群、朋友圈、内容平台等）。
4. 消费者通过推广链路下单后，联盟侧回流订单与佣金数据。
5. 用户在个人中心查看收益状态：预估 / 有效 / 可提现 / 已提现。

### 13.3 平台分成方案（推荐）
采用"平台统一结算 + 用户分账"的模式：
- 联盟佣金先进入平台联盟结算账户；
- 平台按规则分账给用户，并保留平台服务费；
- 首期建议固定分成比例：
  - 用户分成：85%
  - 平台服务费：15%

> 说明：分成比例应后台可配置，为后续分层激励预留空间。

### 13.4 收益口径定义（必须对齐）
1. 预估收益：订单支付后展示，后续可能因退款/维权变化。
2. 有效收益：通过有效性校验后确认。
3. 可提现收益：达到结算周期且满足提现门槛。
4. 已提现收益：打款完成的历史收益。

### 13.5 结算规则（首期建议）
- 结算周期：月结；
- 提现门槛：100 元；
- 退款/维权/无效订单：自动冲减对应预估收益。

### 13.6 对接实施分期
#### Phase 1（MVP）
- 商品池展示
- 专属推广链路生成
- 订单与佣金日级同步
- 收益看板（预估/有效/可提现/已提现）

#### Phase 2（增长）
- 梯度分成（按有效单量或 GMV 提升）
- 类目激励
- 达人等级与更高分成档位

## 14. 战报认证机制（必须）

### 14.1 认证等级
1. 未认证：用户手动填写，仅允许普通展示。
2. 基础认证：截图 + OCR 字段校验（时间、金额、订单量）。
3. 平台认证：基于联盟回流订单自动核验（最高可信）。

### 14.2 展示与分发策略
- 推荐流优先展示"平台认证"战报；
- 未认证内容必须打标"数据未经平台核验"；
- 认证失败需返回原因并支持申诉。

---

## 15. 发帖审核机制（AI + 人审）

### 15.1 审核流程
1. 用户提交帖子；
2. AI 机审实时判定（通过 / 存疑 / 拒绝）；
3. 存疑内容进入人工复审；
4. 输出审核结果与可解释原因。

### 15.2 AI 风险识别范围（首期）
- 诈骗导流、违规外链、灰黑产内容；
- 夸大收益承诺（如"稳赚不赔""保底高收益"）；
- 虚假战报与疑似篡改截图；
- 招聘欺诈与收费陷阱。

### 15.3 审核时效目标（建议）
- 机审响应：< 10 秒；
- 人审处理：工作时段 < 30 分钟。
